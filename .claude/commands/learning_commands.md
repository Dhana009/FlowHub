# Learning & Explanation Commands

Commands for deep-dive learning and concept explanations.

## Concept Explanations
- `@explain [concept]` - Get hybrid explanation (system flow + structured stops)
  - Examples: `@explain Playwright contexts`, `@explain API routing`, `@explain POM`
- `@clarify [concept]` - Get clarification on confusing concept
- `@deep-dive [concept]` - Get senior-level deep dive with internal mechanics

## System Flow Visualization
- `@system-flow` - Visualize UI → API → Backend → DB → API → UI flow
- `@trace-request [scenario]` - Trace a specific request through all layers
- `@show-architecture [project]` - Show architecture for Project 1, 2, or 3

## Mapping Commands
- `@map-to-resume [concept]` - Show how concept maps to resume bullets
- `@map-to-interview [concept]` - Show how concept appears in interviews
- `@map-to-projects [concept]` - Show how concept connects to 3 projects

## Learning Validation
- `@validate-understanding [concept]` - Test understanding with questions
- `@practice-questions [topic]` - Generate practice questions for topic
- `@review-concepts` - Review all concepts learned in current phase
