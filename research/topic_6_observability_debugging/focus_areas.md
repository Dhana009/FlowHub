# Topic 6: Observability & Debugging - Focus Areas

## Where to Focus (Priority-Based)

### CRITICAL Priority (35% Interview Weight - Spend Most Time Here)

1. **Distributed Tracing Concepts & Tools**
   - Jaeger, Zipkin understanding
   - Trace ID propagation
   - Request flow analysis
   - Latency bottleneck identification
   - 40% of study time

2. **Log Analysis & Correlation Patterns**
   - Parsing logs from multiple services
   - Correlating events
   - Request ID correlation
   - Root cause identification
   - 35% of study time

### HIGH Priority (15-20% Interview Weight - Important)

3. **Metrics Collection & SLO Validation**
   - Key metrics understanding
   - Prometheus usage
   - SLO compliance validation
   - Metrics-driven testing
   - 15% of study time

4. **CI/CD Failure Debugging Strategies**
   - Flaky test debugging
   - Log correlation
   - Environmental factor identification
   - Test vs service issue distinction
   - 10% of study time

### MEDIUM Priority (10% Interview Weight - Good to Know)

5. **Service Mesh Observability**
   - Istio, Linkerd understanding
   - Sidecar metrics
   - Traffic pattern analysis
   - 5% of study time

6. **Performance Debugging Techniques**
   - CPU profiling
   - Memory analysis
   - Database query performance
   - Network latency
   - 5% of study time

## What Matters Most

1. **Distributed Tracing** - Most critical (35% interview weight)
2. **Log Analysis & Correlation** - Essential for debugging (30% interview weight)
3. **Metrics & SLO Validation** - Growing importance (20% interview weight)
4. **CI/CD Debugging** - Real-world scenarios (15% interview weight)
5. **Service Mesh** - Modern architecture (10% interview weight)
6. **Performance Debugging** - Advanced topic (10% interview weight)
