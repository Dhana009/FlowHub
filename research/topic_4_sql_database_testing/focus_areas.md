# Topic 4: SQL/Database Testing - Focus Areas

## Where to Focus (Priority-Based)

### Priority 1 (35% Interview Weight - Spend Most Time Here)

1. **Data Isolation & Transaction Control**
   - Most common cause of flaky tests
   - Shows understanding of ACID properties
   - Transaction rollback mechanisms
   - Test data isolation strategies
   - Critical for interview success

### Priority 2 (30% Interview Weight - High Priority)

2. **Writing Validation Queries**
   - Direct application to daily SDET work
   - Tests practical SQL ability
   - SELECT statements for data verification
   - JOIN operations for cross-table validation
   - Data consistency checks

### Priority 3 (20% Interview Weight - Important)

3. **Test Data Setup/Teardown Patterns**
   - Demonstrates test design maturity
   - Environment management
   - Data cleanup strategies
   - Test preconditions validation

### Priority 4 (10% Interview Weight - Good to Know)

4. **Query Optimization for Test Performance**
   - Shows awareness of test execution speed
   - Resource constraints understanding
   - EXPLAIN usage
   - Index strategy

### Priority 5 (5% Interview Weight - Nice to Have)

5. **MongoDB Document Queries**
   - Increasingly relevant
   - Differentiates candidates in modern stacks
   - Document structure validation
   - Array operations

## What Matters Most

1. **Data Isolation & Transaction Control** - Most common interview topic (35%)
2. **Writing Validation Queries** - Practical SQL skills (30%)
3. **Test Data Setup/Teardown** - Test design maturity (20%)
4. **Query Optimization** - Performance awareness (10%)
5. **MongoDB Queries** - Modern stack knowledge (5%)
