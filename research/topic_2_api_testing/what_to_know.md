# Topic 2: API Testing - What to Know

## What Exactly You Need to Know (Interview-Critical Only)

### Core Competencies Tested

1. **Request/Response Validation**
   - Status codes verification
   - Response schema validation
   - Data accuracy checking
   - Data types validation

2. **Authentication & Authorization**
   - API keys management
   - OAuth tokens handling
   - JWT validation and signature verification
   - Security headers understanding

3. **Error Handling & Edge Cases**
   - Timeout handling
   - Rate limits management
   - Error codes (4xx, 5xx) handling
   - Edge case scenarios

4. **Test Design Principles**
   - Setup → Execution → Verification → Reporting → Cleanup pattern
   - Structured approach to API testing
   - Test case design methodology

5. **Integration Capability**
   - Combining API tests with UI automation
   - End-to-end testing approach
   - Integration thinking

6. **Framework Design**
   - Structuring maintainable API test frameworks
   - Scalable test architecture
   - Framework organization

## Core Concepts (What Interviewers Test)

- **HTTP methods** (GET, POST, PUT, PATCH, DELETE)
- **Status codes** (200, 201, 204, 400, 401, 403, 404, 422, 500, 502, 503, 504)
- **JWT token structure** (header.payload.signature)
- **JWT signature verification** (algorithm validation, public key verification)
- **API test design phases** (Setup, Execution, Verification, Reporting, Cleanup)
- **Request/Response validation** (headers, parameters, body schema, content-type)
- **Authentication methods** (API keys, OAuth, JWT)
- **Error handling** (4xx client errors, 5xx server errors)
- **Data-driven testing** (CSV, JSON, database, Excel)
- **API vs UI testing** (differences and integration)
- **Parameter selection challenges** (boundary value analysis, equivalence partitioning)
- **Call sequencing** (order of API calls, dependencies)
- **Schema validation** (JSON schema, response structure)
- **Test data management** (cleanup, cross-environment)
- **Microservices API testing** (distributed systems)
- **Continuous testing integration** (CI/CD pipeline)
