# SDET Coding Round - What to Know

## What Exactly You Need to Know (Interview-Critical Only)

### Core Technical Competencies

1. **Programming Fundamentals**
   - Your chosen language (Python, Java, C#, JavaScript)
   - Object-oriented design principles
   - Design patterns in test context
   - Data structures and algorithms (moderate complexity)
   - Problem-solving under time pressure

2. **Test Automation Framework Architecture**
   - Framework design from scratch
   - Page Object Model (POM) implementation
   - Hybrid framework design
   - Framework variations and trade-offs
   - Scalability considerations

3. **API Testing & HTTP Protocol**
   - REST API testing
   - Request/response validation
   - Status code validation
   - Response schema validation
   - Data integrity checks
   - OAuth 2.0 authentication flows
   - Token refresh and expiration handling
   - Pagination handling

4. **Database Query Basics**
   - SQL queries for test data setup
   - SQL queries for test data verification
   - Database connection handling
   - Transaction management in tests

5. **Test Automation Logic**
   - Selenium/Playwright locator strategies
   - Element interaction patterns
   - Dynamic waits and timing issues
   - Stale element exception handling
   - Exception handling and retry logic
   - Test data management and parameterization

### Language & Framework Expectations

6. **Python (Most Common)**
   - Primary use: Web & desktop automation
   - Frameworks: Pytest, Selenium, PyAutoGUI
   - Most common in SDET roles

7. **Java (Very Common)**
   - Primary use: Enterprise automation
   - Frameworks: TestNG, Selenium, RestAssured
   - Very common in enterprise roles

8. **C# (Common)**
   - Primary use: Windows/enterprise apps
   - Frameworks: NUnit, Selenium, SpecFlow
   - Common in Microsoft ecosystem

9. **JavaScript (Growing)**
   - Primary use: Web automation
   - Frameworks: Cypress, Playwright, Jest
   - Growing adoption (2024-2025)

### Problem Complexity Levels

10. **Easy (20-30% of questions)**
    - String manipulation
    - Basic list operations
    - Simple logic problems

11. **Medium (60-70% of questions)**
    - Framework design
    - Test case generation
    - API response validation
    - Most common complexity level

12. **Hard (10-20% of questions)**
    - Complex automation scenarios
    - Performance optimization
    - Multi-threaded testing
    - Less common but important

### Time Limits & Evaluation

13. **Time Constraints**
    - Typical coding round: 60-90 minutes
    - Usually 1-2 problems
    - Time pressure is part of evaluation

14. **Evaluation Criteria**
    - Correctness: 40%
    - Code quality & design: 30%
    - Communication: 20%
    - Efficiency: 10%

### Python-Specific Coding Skills (For Python Background)

20. **Log Parsing and Analysis**
    - Extract error messages from log files
    - Parse timestamps and correlate events
    - Filter logs by severity or component
    - Generate summary reports from raw logs
    - Handle large log files efficiently
    - Use regex for pattern matching

21. **API Response Validation**
    - Parse nested JSON structures
    - Extract data from paginated responses
    - Compare actual vs expected with tolerance
    - Handle missing/null fields gracefully
    - Transform API data for assertions
    - Validate against JSON schemas

22. **Test Data Generation**
    - Generate unique identifiers (emails, usernames)
    - Create randomized valid test data
    - Build parameterized test data sets
    - Generate mock data matching API schemas
    - Ensure reproducibility with seeding

23. **File Operations**
    - Read/write configuration files (JSON, YAML, CSV)
    - Manage test artifacts and results
    - Clean up temporary files
    - Merge/compare test result files
    - Handle file paths across OS

24. **String and List Manipulation**
    - Parse test method names and extract parameters
    - Manipulate URLs (add/remove query params)
    - Extract and validate emails/identifiers
    - Sort/filter test results
    - Deduplicate while preserving order
    - Transform test output formats

25. **Dictionary Operations**
    - Merge configuration dictionaries with precedence
    - Flatten nested structures
    - Extract configuration sections
    - Build dynamic test parameters
    - Validate configuration completeness

26. **Concurrent Execution**
    - Thread-safe result collection
    - Coordinate parallel test execution
    - Handle race conditions
    - Producer-consumer patterns
    - Thread pool management

27. **Test Utilities**
    - Retry logic with exponential backoff
    - Test result aggregation
    - Compare test execution reports
    - Cache mechanisms for expensive operations
    - Generate test execution summaries

28. **Python-Specific Concepts**
    - List/dictionary comprehensions
    - Regular expressions (re module)
    - File I/O operations
    - Exception handling
    - Functional programming (map, filter)
    - JSON/YAML processing
    - Threading and concurrency
    - Time complexity awareness
    - Type hints (Python 3.10+)
    - Async/await patterns (asyncio)
