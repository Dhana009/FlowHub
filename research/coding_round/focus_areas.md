# SDET Coding Round - Focus Areas

## Where to Focus (Priority-Based)

### Tier 1 - Must Master (70% of Interview Questions)

1. **Test Automation Framework Design**
   - Page Object Model (POM)
   - Hybrid frameworks
   - Framework from scratch
   - Multi-page application handling
   - Most critical topic

2. **Selenium/Playwright Locator Strategies**
   - Element interaction patterns
   - Dynamic locators
   - Locator resilience
   - Critical for UI automation

3. **API Testing with Request/Response Validation**
   - Status code validation
   - Response schema validation
   - Data integrity checks
   - OAuth flows
   - Core SDET skill

4. **Test Data Management and Parameterization**
   - CSV/JSON file reading
   - Database-driven testing
   - Data-driven test design
   - Test data isolation
   - Essential for scalable tests

5. **Exception Handling and Retry Logic**
   - Dynamic waits
   - Stale element handling
   - Retry mechanisms
   - Error handling patterns
   - Critical for reliable tests

### Tier 2 - High Priority (20% of Interview Questions)

6. **Python-Specific Coding (For Python Background)**
   - Log parsing and analysis (Intermediate-Advanced)
   - API response validation (Intermediate)
   - File operations (Beginner-Intermediate)
   - String/list manipulation (Beginner-Intermediate)
   - Dictionary operations (Intermediate)
   - Test utilities (Intermediate-Advanced)
   - Critical for Python coding rounds

7. **SQL Queries for Test Data Setup/Verification**
   - Database queries
   - Test data preparation
   - Data verification
   - Transaction management

8. **JSON/XML Parsing and Assertion**
   - Response parsing
   - Schema validation
   - Data extraction
   - Assertion patterns

9. **Logging and Reporting Mechanisms**
   - Test reporting
   - Logging strategies
   - Failure analysis
   - CI/CD integration

10. **Cross-Browser/Cross-Platform Testing**
    - Multi-browser support
    - Platform considerations
    - Configuration management
    - Less frequent but important

11. **CI/CD Integration Basics**
    - Pipeline integration
    - Test execution in CI
    - Reporting in CI
    - Environment management

### Tier 3 - Nice to Have (10% of Interview Questions)

12. **Python Concurrency (Advanced)**
    - Threading and thread safety
    - Concurrent execution patterns
    - Race condition handling
    - Less common but shows expertise

13. **Performance Testing Concepts**
    - Performance considerations
    - Optimization strategies
    - Less common in coding rounds

14. **Load Testing Frameworks**
    - Load testing basics
    - Tool knowledge
    - Rarely asked in coding rounds

15. **Security Testing Basics**
    - Security considerations
    - Authentication testing
    - Less common

16. **Advanced Design Patterns**
    - Factory, Builder, Singleton
    - In test context
    - Nice to mention

## What Matters Most

1. **Framework Design** - Most critical (70% of questions)
2. **Locator Strategies** - Core UI automation skill
3. **API Testing** - Essential SDET competency
4. **Test Data Management** - Scalability requirement
5. **Exception Handling** - Reliability requirement
6. **Python-Specific Coding** - Critical for Python coding rounds (log parsing, file ops, data manipulation)
7. **SQL Queries** - Database testing (20% of questions)
8. **JSON/XML Parsing** - API testing requirement
9. **Logging/Reporting** - Professional code quality

## Time Investment Summary

- **Tier 1 (Must Master)**: 70% of prep time
- **Tier 2 (High Priority)**: 20% of prep time
- **Tier 3 (Nice to Have)**: 10% of prep time

## Problem Complexity Focus

- **Medium complexity**: 60-70% of questions (focus here)
- **Easy complexity**: 20-30% of questions (quick review)
- **Hard complexity**: 10-20% of questions (understand concepts)
